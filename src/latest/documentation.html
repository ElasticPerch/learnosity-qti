Mapping Items and Questions

Both Learnosity item and questions are mapped to Assessment Item

Mapping Rules 101 for Item export

  • Assessment Item identifier is taken from Item `reference`, however if it's QTI invalid then a random one would be generated and previous Learnosity Item `reference` will be stored within `label` attribute.
  • Questions within Item are mapped to QTI Interactions. Their identifiers are taken from Question `reference`, however if QTI invalid, a random identifier would be generated and the original `reference` will be stored within interaction `label` attributes.
  • Some feature and inline features are mapped to QTI `object` element and their `reference`(s) are discarded. Current supported feature types:
    • audioplayer
    • videoplayer

Mapping Rules 101 for Question export

  • Assessment Item identifier is taken from Question `reference`, however if it's invalid then a random one would be generated and previous Learnosity Item `reference` will be stored as label.
  • Mapping widget type feature is not supported and it will fail.

Other Mapping Rules 101

  • Div block might be added to wrap around HTML elements when appropriate. For example, QTI itembody only allow Block elements within its content and if original HTML content from Learnosty Item `content` has spans, the transformation library would try its best to wrap it with a div block to make it a valid QTI.
  • QTI response declaration object is vastly different than Learnosity. Thus, `alt_responses` and `penalty` are currently ignored.
  • We only map our validation object to QTI pre-defined response processing templates, its either `map_response` or `match_correct`. If our validation object within Item and its respective questions only contain a score of 1, then it would always be mapped to `match_correct`. Everything else, we would map it to `map_response` response processing template.

Mapping Assessment Items

An `assessmentItem` element in QTI is generally mapped one to one to a Learnosity Item and its `interactions` are mapped to Learnosity Question(s). To learn more about Item and Question in Learnosity context please read our documentation: http://docs.learnosity.com/tutorials/tutorial_101#intro

An Item can be represented in a JSON format with below attributes.

Attribute Mandatory Type Notes 
reference
 
Y
 
string
 A unique item identifier across Itembank. It has a limitation of maximum of 150 characters and must only contain ASCII printable characters, except for double quotes, single quotes and accent. 
questionReferences
 
Y
 
array
 An array of question references. Like item reference, it has a limitation of maximum of 150 characters and must only contain ASCII printable characters, except for double quotes, single quotes and accent. ie. `questionReferenceOne` 
content
 
Y
 
string
 A valid HTML string that may contains the structure that holds Questions. It shall include questions placeholder set in `questionReferences`, ie. <span class="learnosity-response question-questionReferenceOne"></span>

N
 
string
 ie. `published`, `unpublished`, and `deleted`

N
 
string

N

N
 
array

Mapping Rules 101

  • Item `reference` is taken from <assessmentItem> identifier
  • Question `reference` would be mapped from the combination of <assessmentItem> and interaction identifier separated by underscore
  • Status is mapped as `published`
  • Generally <itemBody> content would have its <rubricBlock> stripped and be mapped to first Question's `stimulus` with interaction elements replaced with Question `span` placeholder.
  • Special case for an assessment item with only either `inlineChoiceInteraction` or `textEntryInteraction` types, the entire <itemBody> content would be mapped to Question `template`. See example on `inlineChoiceInteraction` below.
  • Learnosity generally doesn't accept <object> elements, therefore some conversion will take place:
    • Object with mime type of `image/...` will be converted to a regular <img> tag
    • Object with mime type of `audio/...` will be converted to an inline Learnosity `audioplayer` feature (https://docs.learnosity.com/assessment/questions/featuretypes?renderWidgets=true#audioplayer)
    • Object with mime type of `video/...` will be converted to an inline Learnosity `videoplayer` feature (https://docs.learnosity.com/assessment/questions/featuretypes?renderWidgets=true#videoplayer)

Easily import these items and questions JSON to Learnosity Itembank using Data API. See documentation at http://docs.learnosity.com/reporting/data/initialisation

Examples

An example of mapping a QTI `choiceInteraction` assessment item (choice.xml) at http://www.imsglobal.org/question/qtiv2p1/examples/items/choice.xml to Learnosity Item and Question with `mcq` question type. 

{
    "reference": "choice",
    "questionReferences": [
        "choice_RESPONSE"
    ],
    "content": "<span class=\"learnosity-response question-choice_RESPONSE\"></span>",
    "status": "published",
    "description": "Unattended Luggage"
}
[
    {
        "reference": "choice_RESPONSE",
        "type": "mcq",
        "data": {
            "stimulus": "<p>Look at the text in the picture.</p><p><img src=\"images/sign.png\" alt=\"NEVER LEAVE LUGGAGE UNATTENDED\"/></p>What does it say?",
            "type": "mcq",
            "options": [
                {
                    "label": "You must stay with your luggage at all times.",
                    "value": "ChoiceA"
                },
                {
                    "label": "Do not let someone else look after your luggage.",
                    "value": "ChoiceB"
                },
                {
                    "label": "Remember your luggage when you leave.",
                    "value": "ChoiceC"
                }
            ],
            "shuffle_options": false
        },
        "widget_type": "response"
    }
]
